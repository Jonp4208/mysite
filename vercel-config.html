Vercel Configuration Instructions

It seems that Vercel is still using an old configuration. Let's configure the deployment directly through the Vercel dashboard:

Step 1: Go to the Vercel Dashboard

First, go to the Vercel dashboard and select your project:

Open Vercel Dashboard

Step 2: Access Project Settings

Click on your project, then click on the "Settings" tab at the top of the page.

Step 3: Configure Build & Development Settings

In the Settings page, find and click on "Build & Development Settings" in the left sidebar.

Update the following settings:

  • Framework Preset: Next.js
  • Build Command: next build
  • Output Directory: .next
  • Install Command: npm install
  • Development Command: next dev

Click "Save" when you're done.

Step 4: Configure Environment Variables

In the Settings page, find and click on "Environment Variables" in the left sidebar.

Add the following environment variable:

  • Name: DISABLE_PWA
  • Value: true

Make sure to select all environments (Production, Preview, and Development).

Click "Save" when you're done.

Step 5: Redeploy Your Project

Go back to the "Deployments" tab and click on the "..." menu next to your latest deployment.

Select "Redeploy" from the dropdown menu.

This will trigger a new deployment with the updated settings.

Alternative Approach: Use Vercel CLI

If you prefer to deploy using the Vercel CLI, you can run the following commands:

  1. Install Vercel CLI: npm install -g vercel
  2. Login to Vercel: vercel login
  3. Deploy your project: vercel --prod

Follow the prompts to configure your project.

After following these steps, your Next.js project should deploy successfully on Vercel.
